# Инструкции для Claude Code — funding-service

## Старт каждой сессии (ОБЯЗАТЕЛЬНО)

После `/clear` контекст сбрасывается полностью. Перед выполнением любой задачи:

1. Прочитать `C:\projects\obsidianVaults\defaultVault\funding\00-home\index.md` — текущее состояние
2. Прочитать `C:\projects\obsidianVaults\defaultVault\funding\00-home\priorities.md` — следующая задача и backlog
3. Прочитать нужный раздел `funding-service-plan.md` — полное описание задачи и критерий готовности
4. При необходимости заглянуть в `atlas/architecture.md` и `atlas/stack.md`

Без этого шага браться за код нельзя — контекст потерян после `/clear`.

## Правила работы над проектом

### Коммиты

После завершения каждой задачи из плана (`funding-service-plan.md`) делать git commit.  
Формат сообщения: `feat: <описание> (task X.Y)`.  
Коммитить только по явному подтверждению пользователя или после завершения задачи.

### Документация и аналитика

После каждой выполненной задачи обновлять vault в:
```
C:\projects\obsidianVaults\defaultVault\funding\
```

Структура vault (по образцу `copilka`):
```
funding/
├── 00-home/
│   ├── index.md        — текущее состояние проекта
│   └── priorities.md   — backlog с чекбоксами задач
├── atlas/
│   ├── architecture.md — структура проекта и слои
│   └── stack.md        — технологический стек
├── sessions/
│   └── YYYY-MM-DD-claude-N-slug.md  — лог каждой сессии
└── knowledge/
    ├── decisions/      — архитектурные решения
    └── patterns/       — переиспользуемые паттерны
```

**После каждой задачи:**
1. Создать файл сессии в `sessions/` с коммитом и описанием что сделано
2. Обновить чекбокс в `priorities.md`
3. Обновить `index.md` (раздел «Что работает» и «Сессии»)

### Порядок выполнения задач

Строго следовать плану из `funding-service/` (рядом с этим файлом лежит `funding-service-plan.md`).  
Одна задача — один промпт — один коммит.

### Тесты

Каждая задача содержит критерий готовности. Не считать задачу выполненной, пока критерий не достигнут.  
Тесты пишутся вместе с кодом, не отдельно.

### Секреты

`.env` не коммитить никогда. В репо только `.env.example`.

### Структура проекта

Весь код сервиса находится в `funding-service/`.  
Корень репозитория (`C:\projects\fanding\fanding\`) содержит только:
- `funding-service/` — сам проект
- `funding-service-plan.md` — план разработки
- `CLAUDE.md` — этот файл
